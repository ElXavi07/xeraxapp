# Security Policy

Xerax Root coordinates operations that can erase data or make a device
unbootable. We welcome responsible disclosure of vulnerabilities involving the
local companion, browser pairing, command validation, image handling, update
delivery, workflow manifests, or sensitive data.

## Reporting

Do not open a public issue for an exploitable vulnerability. Use GitHub's private
vulnerability reporting feature for this repository. Include affected versions,
reproduction steps, impact, and any suggested mitigation.

Do not include real pairing tokens, device serial numbers, API keys, personal
files, or proprietary firmware in a report.

## Scope

Security boundaries and intentional limitations are documented in
[`docs/architecture.md`](docs/architecture.md). Reports that demonstrate a bypass
of those boundaries are particularly valuable.
