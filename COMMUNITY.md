# Join The Xerax Community

Xerax is for people who believe powerful Android tools can also be transparent,
reviewable, and honest about risk.

## You Can Help Without Writing Code

- Request a device using its exact model and firmware information.
- Test inspection-only builds on hardware you own.
- Improve beginner-facing explanations.
- Translate the interface and documentation.
- Design terminal states, recovery screens, and accessibility improvements.
- Review privacy language and consent flows.
- Share the project with responsible Android communities.

## Technical Contribution Paths

- **Android researchers:** document partitions, boot behavior, and recovery.
- **Python developers:** improve the companion, validators, and workflow engine.
- **Frontend developers:** build the terminal, progress timeline, and reports.
- **Security engineers:** review pairing, updates, artifacts, and command gates.
- **Test engineers:** create fixtures and replayable failure cases.
- **Technical writers:** turn difficult device procedures into clear guidance.
- **AI engineers:** build bounded structured-output planning and evaluations.

## What Makes Evidence Useful

A strong device report identifies:

- Manufacturer, exact model, and product codename.
- Android version and firmware fingerprint.
- Security patch level.
- Bootloader policy and unlock result.
- Slot and partition layout.
- Root method and exact artifact hashes.
- Sanitized command results.
- Reboot, verification, and recovery outcome.

Never include serial numbers, personal files, credentials, pairing tokens, API
keys, or proprietary firmware.

## Recognition

The project intends to recognize meaningful work through:

- Contributor credits.
- Device-profile author and verifier attribution.
- Founding supporter recognition.
- Public milestone acknowledgements.
- Maintainer roles earned through sustained, careful contributions.
- Sponsored device-testing reports.

Recognition never replaces review. A highly trusted contributor still needs
evidence for a device workflow.

## Supporting The Project

Stars, issue reports, testing, documentation, and code are valuable support right
now. Future financial sponsorship will be directed toward:

- Physical test devices.
- Code-signing certificates.
- OpenAI API usage for automation testing.
- Build and download infrastructure.
- Firmware research.
- Accessibility and localization work.
- Independent security audits.

Funding goals and completed spending should be reported publicly before formal
sponsorship is introduced.

## Community Standard

Be patient with beginners, precise about technical claims, and direct about risk.
Do not shame users for mistakes or encourage experiments that lack a recovery
path. This project exists to make Android modification more understandable, not
more reckless.

Start with [CONTRIBUTING.md](CONTRIBUTING.md), then open a focused issue or pull
request.
